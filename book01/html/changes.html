Adventures in JavaScript

Changes

These changes should not concern you unless you started playing the game and I needed to update things out from under you. If you were in the middle of the game and things changed, this change list can help you get aligned again.

20 January 2014

Change _player.platform (stage 2)
Previously, _player.platform started out as a object, but that has been changed to have it start out as a boolean. It is changed into an object later (in stage 3) when a simple true/false value is not sufficient to support multiple platforms. This change was made to make the early code edits simpler and easier to understand.

14 January 2014

Move platform.origin_y and origin_y init later (stage 1)
Originally, stage 1 node 003 started out by initializing platforms with origin_x and origin_y attributes (without an explanation why they were needed). These attributes have been moved into stage2 which is where the changes are needed to allow the collide() function to be used.
Nodes updated:
stage 1: 003
stage 2: 027, 046

Fix max falling velocity (stage 2)
Previous code did not enforce max falling velocity. New code does.
Nodes updated:
stage 2: 025

Fix jumping (stage 2)
Old code allowed you to jump in mid-air if you fell off the edge of a platform. New code fixes that by replacing the is_jumping flag with a variable that records the platform that the player is on.
Nodes updated:
stage 2: 028, 032, 033, (context only: 046)
stage 3: 073, 075

Draw monsters in front of goals (stage 3)
Monsters used to be drawn before the goal, which meant that if they overlapped the monsters would go behind the goal image. This has been switched so that the monsters are drawn in front of the goals.
Nodes updated:
stage 3: 065

Add "Transition I - Levels" badge (stage 4)
The "Transition I - Levels" badge has been added to the end of stage 4. If you finished Stage 4, then you have already earned this badge.
Nodes updated:
stage 4: 093

Init player start (x,y) (stage 4)
Init player_start_x and player_start_y in init_level_defaults. This inits it properly for the title screen later.
Nodes updated:
stage 4: 091

Better names for data arrays (stage 4)
Use monster_data instead of monst for the array of monsters. Similarly with item_data instead of items.
