About

Adventures in JavaScript is a "branching path book" that shows you how to write a canvas-based JavaScript game (a classic platform-jumping game) during the course of the adventure.

Unlike most branching path books, there are no wrong paths in this book. The choices merely determine the order in which you perform the various tasks necessary for the game. Two adventurers who choose different paths through this book will still end up in the same place at the end - they will both have created the same sample game.

This book is intended to serve as a jumping off point so that students can continue to develop their programming skills by modifying the levels and adding new ones.

Currently, this book is in beta, and it also assumes that the student has already completed a simple "baseline" project where they were introduced to:

  • HTML
  • CSS
  • Adding JavaScript to a webpage
  • Variables and functions
  • Using the browser's inspector to debug web pages and view the console log
  • Creating a canvas and drawing on it from JavaScript
  • Using the arrow keys (and WASD) to move an object around the screen

However, if you have any familiarity with HTML and/or JavaScript, you can probably just jump in without having any problems. Just grab a copy of the baseline project to get started.

Requirements

To play the game (and learn how to program a game), you will need the following:

  • A web browser. Something modern.
  • A text editor. Modern or otherwise.
  • A copy of the baseline project.
  • The sample images.
  • A character sheet to track your progress†.
  • A map for Stage 5 (assuming you accept the challenge token).

If you plan on making your own images (and you should!) you may also want to track down an image editor. Nothing fancy is needed and there are a lot of "icon editors" that are perfect for drawing the small sprite images used in these sort of games.

Current status

The game is now 100% complete, but the text is still a painfully early draft. There are many placeholders where a section needs to be expanded with more information.

However, all of the programming parts are in working order. The build process for this project walks through every possible pathway in the adventure and performs the code edits to ensure that everything works as intended. At the end of each stage, a check is made to verify that every pathway through that stage produces the exact same code.

Be aware that, while the code parts that are done are reasonably stable, this project is still in beta, so some tweaks to the code may take place. This would only cause problems if you were part-way through the adventure when these changes were made. Whenever changes like this are made, they will be noted in a changelog.

Development info

To get an idea of how the final game plays, you can take a look at a prototype of the finished game. The code in the final version will be slightly different as the code gets chopped up into bite-sized pieces, but the overall gameplay will be basically the same.

For those who are curious, there are 2 development maps (one for stages 1-4 and one for stage 5) that show the overall structure of the game. These maps are not needed while playing.

† - Character sheet based on Dungeon World by Sage LaTorra and Adam Koebel.
